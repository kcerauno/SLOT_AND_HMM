# アイディアノート：HMM 派生スロット文法 — V8 文法の精緻化修正版の可能性

作成日: 2026-03-08
根拠分析: [analysis_report.md](analysis_report.md)（E-1〜E-3 の統合考察）

---

## 1. 動機：なぜこのアイディアが生まれたか

hypothesis/05〜09 の一連の分析から、以下の事実が確立されている。

- **HMM（特に Trigram）は複合語境界を統計的に学習している**（E-1: 境界位置効果 有意、E-3: 語集合効果 有意）
- B-start 状態（S2: gallows/c 系、S5: y/v 系）と B-end 状態は一貫して異なる状態が担い、**方向性のある境界マーキング**が存在する（hypothesis/06 提案 B）
- 基内位置勾配（hypothesis/07）では head 集中・tail 集中の状態が明確に分離しており、HMM は「基の内部構造」を暗黙に学習している

これらは、HMM が V8 スロット文法と類似した「文字クラスの順序制約」を データから学習していることを示唆する。

**問い：HMM の学習内容を逆引きして、V8 文法を精緻化することができないか？**

---

## 2. V8 スロット文法と HMM の構造的対応と差異

### 2.1 対応している部分

| V8 スロット文法 | HMM の対応する学習内容 |
|---------------|---------------------|
| 各スロットの文字クラス定義 | 各状態の放射分布 B[state, char] |
| 基内の左→右スロット順序制約 | 状態遷移確率 A[i, j]（左→右バイアス）|
| 基の先頭文字クラス（slot 1〜3）| B-start 状態の放射分布（hypothesis/06 提案 A）|
| 基の末尾文字クラス（slot 14〜15）| B-end 状態の放射分布（hypothesis/06 提案 B）|
| 基内の中間文字クラス（slot 4〜13）| 基内中間位置での状態分布（hypothesis/07 提案 D）|

### 2.2 解決されない根本的差異

**V8 はポジション型テンプレート、HMM は文脈依存型**

```
V8 スロット文法:
  slot 1（基の1文字目）→ slot 2（2文字目）→ ... → slot 16（最大16文字目）
  ※ 「基の中の何番目か」という絶対位置が定義の基本

HMM:
  P(s_t | s_{t-2}, s_{t-1}) × P(c_t | s_t)
  ※ 文脈（直前の状態）に基づくが、「基の中の何番目か」は知らない
```

HMM の状態は「基の中の位置」ではなく「直前の状態から見た文脈」で決まる。状態 S_i が「slot 9 相当の文字を放射する」としても、その状態が基の何番目に現れるかは確率的に変動する。

### 2.3 k=16〜20 での実施について

状態数を V8 スロット数（16）に合わせることで**放射分布の粒度**は改善するが、**位置ミスマッチ**は解決しない。

| 改善できる問題 | 改善できない問題 |
|-------------|----------------|
| 文字クラスの粒度（k=7〜8 では複数スロットが1状態に圧縮）| 「基内何番目か」という位置情報の欠如 |
| 各スロット相当の文字群の分離 | 1対1 スロット↔状態対応の保証 |
| B-start/B-end の文字クラスの細分化 | 複数のスロットが1状態に分裂する可能性 |

k=16〜20 を実施する場合は、compound_words.txt の分割情報を**外部ラベル**として用いた後付け位置分析が必要になる（次節参照）。

## 3. 提案アプローチ：3 ステップの HMM 派生スロット文法構築

### ステップ 1：放射分布からスロット候補文字クラスを同定

compound_words.txt の各基内の文字に対して Viterbi デコードを実施し、**基内位置（先頭から何番目か）× 状態** のクロス集計表を作成する。

```
基内位置 1 での状態分布 → 「基の先頭に来る文字クラス」
基内位置 2 での状態分布 → 「基の2文字目に来る文字クラス」
...
```

各位置で最頻出の状態の放射分布が、その位置に対応する「HMM 由来のスロット定義」となる。

**期待される出力（例）：**
```
位置 1（基先頭）: S2 が最頻 → 放射: {c: 0.59, a: 0.19, m: 0.11, ...}
                  → HMM-slot-1: {c, a, m, ch-like}
位置 末尾:        S5 が最頻 → 放射: {y: 0.64, v: 0.23, ...}
                  → HMM-slot-tail: {y, v, g}
```

### ステップ 2：V8 スロット定義との照合・差分抽出

HMM 由来のスロット定義と V8 の定義を照合し、以下を特定する：

| 照合結果 | 意味 | 対処 |
|---------|------|------|
| HMM 文字クラス ⊂ V8 スロット | V8 が過剰に広い → 制限の余地 | 該当文字をスロットから削除候補 |
| HMM 文字クラス ⊃ V8 スロット | V8 が過剰に狭い → 拡張の余地 | 不足文字をスロットに追加候補 |
| 位置の不一致 | V8 の順序が HMM と合わない | スロット番号の見直し候補 |
| 一致 | 確信度が高い部分 | 現行 V8 定義を支持するエビデンス |

### ステップ 3：修正スロット定義の提案と再検証

差分分析から得られた修正案を新たなスロット定義として実装し、E-2 相当の文法バリアント分析（e2_grammar_variants_greedy.py）で境界シグナルの変化を測定する。

シグナルが強化されれば、修正が V8 文法の精緻化として有効であることが統計的に確認できる。

## 5. 実施上の注意点

### 5.1 局所最適解の問題

Baum-Welch（EM）アルゴリズムは局所最適解に収束するため、同一の k でも初期化により異なる状態割り当てが生まれる。本アプローチでは：
- 複数回の独立学習からの結果を比較する
- V8 スロット文字クラスを放射分布の初期値として与える（V8 誘導初期化）方法が有効かもしれない

### 5.2 統計量の解釈

HMM は確率モデルであるため、「スロット X = 状態 S_i の文字クラス」という対応は**確率的・統計的**なものにとどまる。実装するスロット文法は hard な yes/no 制約になるため、どこかに閾値判断が必要になる。

### 5.3 k の選択と phantom 状態

k=16〜20 では phantom 状態（Viterbi 占有率 0%）が増加する可能性がある。有効状態数が実質 10〜12 程度になれば、16 スロットへの完全マッピングは困難になる。まず k=12 程度から試みるのが現実的かもしれない。

## 6. 想定される成果

実施できた場合に得られる成果：

1. **V8 スロット定義の経験的検証** — データが支持するスロット文字クラスと、現行 V8 との一致・不一致の定量化
2. **データ駆動スロット文法 V9** — HMM の学習内容を反映した修正スロット定義
3. **境界シグナル強度の改善** — E-2 で V3 がシグナルを強化したように、精緻化された文法が境界シグナルをさらに強化する可能性
4. **「HMM が学習した文法」と「人間設計の文法」の比較** — どこが一致し、どこが異なるかの体系的記述
