## 0. hypothesis/08 からの修正点

### 修正の経緯

hypothesis/08 の E-2 スクリプト（`e2_grammar_variants.py`）は、`find_splits_variant` 関数で「最短先頭基優先（first-fit）」アルゴリズムを使用していた。これは compound_words.txt の生成元である `parse_v4b.py` の「貪欲（greedy）」アルゴリズムと異なる。

**バグの具体例：**

| 語 | parse_v4b.py（貪欲）| find_splits_variant（first-fit）|
|---|---------------------|-------------------------------|
| ackaldy | a + **ck** + al + dy | a + **c** + ka + ldy |
| adal    | **ad** + al | **a** + dal |
| adam    | **ad** + am | **a** + dam |

`ck`（slot 9 + slot 15）は貪欲で正しく認識されるが、first-fit では `c`（slot 9 の先頭）だけで終了してしまう。

**結果への影響：**

- hypothesis/08 の V0_baseline は compound_words.txt と同一語集合を対象にしながら、分割点は **3,351 語（99.6%）で異なっていた**
- このため V0_baseline の Focus State は S4（k=7）で B-start < S-head の「逆転」となり、hypothesis/05 の結果（Focus S2、B-start >> S-head）と不一致だった
- hypothesis/08 のこの結果は **アルゴリズムバグに起因する人工産物**であり、V4/SLOTS_V8 間の真の差異を反映していない

### 修正内容

hypothesis/09（本ディレクトリ）では E-2 を `parse_v4b.py` の `parse_base_greedy` と等価な貪欲アルゴリズムで再実装し、V0_baseline が compound_words.txt と完全一致することを検証した。

```
V0_baseline 検証結果:
  元 compound_words.txt: 3,363 語
  V0_baseline (貪欲)   : 3,363 語
  同一分割             : 3,363 語（100%）
  分割点が異なる       : 0 語
```

## 1. 目的と問い

本分析は次の問いに答えることを目的とする。

> **「hypothesis/05 で確認された複合語境界シグナルは V8 文法定義に依存するか？**
> それとも、文法によらない頑健な統計的実在として存在するか？」

この問いを 3 つのサブ問いに分解して検証した。

## 2. E-1: 置換検定 — 境界位置は本物か

### 2.1 手法の設計

compound_words.txt の 3,363 語の「語」は固定したまま、**各語内の境界位置のみをランダム化**（基数固定）した帰無分布を 1,000 回構築した。統計量には χ²（2×2 分割表 B-start vs S-head）を採用した。
→ 初回は −log₁₀(Fisher p) を使用したが trigram k=8 で float フロア(p=0)によるキャップ飽和が発生し、chi² に修正した。

### 2.2 結果

|--------|---|------------|--------|---------|----------|---------|------|
| trigram | 7 | S2 | **736.5** | 472.0 | **1.56×** | **< 0.001** | H₀ 棄却 |
| trigram | 8 | S5 | **1701.7** | 1384.6 | **1.23×** | **< 0.001** | H₀ 棄却 |
| bigram  | 7 | S2 | 1169.7 | 1116.5 | 1.05× | 0.077 | 棄却失敗 |
| bigram  | 8 | S1 | **1227.5** | 1101.0 | **1.12×** | **< 0.001** | H₀ 棄却 |

### 2.3 考察

**3/4 モデルで H₀ 棄却（p < 0.001）**
V4 文法が定義した実際の境界位置は、同じ複合語内のランダムな位置よりも有意に大きな B-start 特化シグナルを生み出している。V4 文法の複合語分割アルゴリズムが、HMM の状態遷移パターンと整合した言語的に有意な境界を捉えていることを示す。

**Bigram k=7 の例外（p = 0.077）**
帰無中央値（1116.5）と観測値（1169.7）の差はわずか 4.8%（他モデルは 12〜56%）。Bigram k=7 の Focus State S2 は a/y/o 系母音を放射するため、複合語内の任意の母音位置が境界としてラベリングされれば同程度の偏りが生じる。このモデルでは、境界シグナルの一部は「複合語語集合の選択」に依存しており、「境界の正確な位置指定」への依存は弱い。

**Trigram の優位性**
trigram k=7 の観測/帰無比 1.56× は他の 3 モデル（1.05〜1.23×）を大きく上回る。s_{t-2} の追加文脈が境界位置の「正確さ」をより鋭く識別していることを示す。

### 3.3 考察：SLOTS_V8 小幅変更への完全な頑健性

**全バリアントで同じ Focus State + 同じ方向性**
V0〜V3 のすべてのバリアントで Focus State が S2（k=7）/ S5（k=8）と完全に一致し、B-start >> S-head の方向性も変わらない。Fisher p 値も全バリアントで極めて高い有意性を維持している。

## 4. E-3: 語集合シャッフル検定 — 複合語集合の選択は特別か

### 4.1 手法の設計

E-1 では「同じ複合語の中で境界位置だけをランダム化」したが、「そもそも複合語という語集合自体が HMM 的に特殊な語群なのか」は検定できていなかった。E-3 はこの問いを検定するため、単独語プール（4,695 語）から 3,363 語をランダムに復元抽出し、compound_words.txt の基数分布に従ってランダム境界を割り当てた帰無分布を 1,000 回構築した。

- **S-head の参照**: E-1 と同一（実際の単独語先頭状態）
- **B-start の帰無**: ランダムサンプリングした単独語 × ランダム境界位置

### 4.4 考察

**Trigram k=8 の 778× という異常値**
trigram k=8 の帰無中央値はわずか 2.2 であり、単独語プールからランダムに選んだ語では S5（y/v 系）への B-start 特化がほぼ生じない。一方、実際の複合語集合では χ² = 1701.7 と極めて大きな値を示す。これは S5 が「複合語という語クラスの境界先頭」に高度に特化した状態であり、単独語の任意位置では再現できないことを意味する。すなわち **S5 の B-start 特化は複合語集合の本質的な性質**である。

**Bigram k=7 の二重の棄却失敗**
E-1（境界位置）と E-3（語集合）の両方で棄却失敗となった bigram k=7 は、「複合語の境界位置を正確に指定しても、語集合として複合語を選んでも」シグナルが変わらない。これは Focus S2（a/y/o 系母音放射）がヴォイニッチ語彙全体に広く分布しており、複合語・単独語・境界・非境界の区別に依存しない状態であることを示す。このモデルでのシグナルは HMM の放射分布の構造に由来する。

**「境界シグナルの実在」に対する E-3 の意味**
E-3 が有意（trigram k=7/k=8、bigram k=8）の場合、複合語集合は単独語プールと比べて HMM 的に異質な語群であることが示される。これは複合語として分類された語が実際に構造的に異なる（語内パターンが単独語と異なる）可能性を支持する。

## 6. 追加考察

### 6.1 V4 文法の言語的妥当性の再評価

hypothesis/08 では「SLOTS_V8 と V4 は 99.6% の語で分割点が異なる」という発見が、HMM の学習した境界概念が V4 文法に近いという仮説を示唆していた。しかしこれは first-fit バグに由来する誤った分割であり、SLOTS_V8 の貪欲分割（V0_baseline）と V4 の貪欲分割（compound_words.txt）は同一であることが確認された。

言い換えると、**parse_v4b.py のスロット定義（= SLOTS_V8）と貪欲アルゴリズムが compound_words.txt の唯一の生成源**であり、hypothesis/05 の結果は SLOTS_V8 文法の正しい実装から直接導かれる。

### 6.2 E-3 は既存結論を否定するか、肯定するか

**結論：E-3 は既存結論を「否定」しない。「精緻化・補強」する。**

既存の主要結論は「境界シグナルは境界位置の正確さに依存し（E-1）、文法小幅変更に頑健（E-2）」であった。E-3 はこれに加えて「複合語集合の選択自体も独立したシグナル源である」という新たな事実を追加する。

具体的な影響を結論別に整理する：

| 既存結論 | E-3 との関係 | 理由 |
|---------|------------|------|
| **1. 境界シグナルの実在（E-1）** | **補強** | E-3 でも 3/4 モデルで有意 → 実在性の証拠が複数の独立な手法で確認された |
| **2. SLOTS_V8 小幅変更への頑健性（E-2）** | **影響なし** | E-3 は語集合の問いであり、E-2 の文法定義感度とは直交する検定 |
| **3. hypothesis/08「逆転」の訂正** | **影響なし** | バグ由来の人工産物という事実は E-3 によって変わらない |
| **4. 主要判断：境界シグナルは頑健** | **精緻化** | 「シグナルは境界位置 + 語集合の両方に依存する」と解像度が上がる |

**特に重要な精緻化：Bigram k=7 の完全説明**

Bigram k=7 は E-1 でも E-3 でも棄却失敗となった唯一のモデルである。これにより、このモデルの B-start シグナルは「境界位置の正確さにも、複合語集合の特殊性にも」依存しないことが確定した。Focus S2 が a/y/o 系母音を広く放射し、ヴォイニッチ語彙全体で頻繁な状態であるため、どの語のどの位置を境界としてラベリングしても同程度のシグナルが生まれる。これは既存の考察（「語集合選択依存型」という表現）を「HMM 状態空間構造のみに由来」と訂正・精緻化するものである。

**Trigram k=8 の新発見**

E-3 の 778× という数値は、trigram k=8 の S5 特化が「複合語集合の本質的な性質」であることを示す、既存分析では得られなかった知見である。E-1 のみでは「境界位置が正しければシグナルが出る」とだけ言えたが、E-3 により「複合語という語クラス自体が S5 を呼び出す性質を持つ」という踏み込んだ解釈が可能になった。

### 6.3 Bigram k=7 S2 が学習しているもの：語頭母音優位性

E-1・E-3 の双方で棄却失敗となった bigram k=7 のシグナルが「HMM 状態空間構造のみ」に由来するとした場合、それは V4/V8 文法に対応する複合語境界の概念ではなく、**別の何かを HMM が学習している**ことを意味する。その正体を以下に考察する。

**数値から見た S2 の普遍性**

| 比較対象 | χ² | 観測値との差 |
|---------|-----|-----------|
| 実観測（複合語境界）| 1169.7 | — |
| E-1 帰無中央値（同語内ランダム境界）| 1116.5 | −4.8% |
| E-3 帰無中央値（単独語ランダム選択＋ランダム境界）| 1133.7 | −3.2% |

実観測との差が 3〜5% しかない。これは S2 が「複合語境界先頭」のみならず「任意の語の任意の位置」でほぼ同頻度に呼び出されることを意味する。

**S2 が学習しているパターンの正体**

hypothesis/06 の放射分析から S2 は **a / y / o 系母音を高確率で放射する**状態であることが確認されている。ヴォイニッチ語では：

- 単独語先頭（S-head）にも母音 a/y/o が高頻度で現れる
- 複合語の各基先頭（B-start）にも同様に母音が現れる
- 複合語内のランダム位置にも母音は一定頻度で現れる

つまり S2 が捉えているのは「**ヴォイニッチ語彙全体に通底する語頭母音優位性（vowel-initial preference）**」であり、複合語境界に固有の現象ではない。V4/V8 文法が定義する「基の区切り」という概念よりも上位の、語彙全体の graphotactic 制約を反映している。

**Bigram が区別できず Trigram が区別できる理由**

Trigram k=7 の Focus S2 は「c / a / m 放射（gallows 系）」であり、境界位置（E-1: 1.56×、E-3: 1.51×）の両方で有意なシグナルを示す。Trigram は s_{t-2} という2ステップ前の状態を文脈として持つため、「基の末尾付近の状態連鎖」が境界の予兆として機能する。これにより「単独語先頭」と「複合語内基先頭」の手前の文脈を区別できる。

一方、Bigram は s_{t-1}（直前の状態）しか参照できない。基末端の1文字直後という局所的な文脈だけでは、「単独語先頭の直前（= 文中での語境界）」と「複合語内基境界の直前」を区別するのに十分な情報が得られない。

**予想される含意**

Bigram k=7 の結果は、ヴォイニッチ語の以下の構造的特徴を示唆する：

1. **複合語の各基と単独語は、先頭文字の分布がほぼ同一**である（どちらも母音優位）
2. **複合語境界の識別には少なくとも2ステップ以上の文脈（bigram では不足）が必要**
3. 単独基（単独語）と複合語の各基は、先頭文字から見た形では「同じ語頭文字分布に従う単位」として HMM に学習されている可能性がある

これは逆説的だが「複合語の基は、それ単体で見れば単独語と形態的に区別できない」という言語学的仮説を支持するエビデンスとも解釈できる。複合語は「単独語として成立する単位の連結」であるという V8 文法の設計思想と整合する。

### 6.4 SLOTS_V8 文法の拡張方向性

E-2 の V3（slot10 に 'e' 追加）でシグナルが強化された現象は、スロット定義の拡張が境界シグナルに与える影響を示す好例である。slot10（V_a2）の語彙拡張により、一部の語で複合語数が減少する（-134 語）一方、残った複合語の境界シグナルがより明確になった。これは文法の精緻化がシグナル品質を向上させる可能性を示唆する。

## 7. 結論（E-3 追記版）

1. **境界シグナルの実在性（E-1 から。E-3 で補強）**
   V4/SLOTS_V8 文法の複合語境界位置は、同語内のランダム位置より有意に大きな B-start 特化シグナルを持つ（4 モデル中 3 モデルで p < 0.001）。E-3 でも同様に 3/4 モデルで H₀ 棄却が得られ、独立した手法から実在性が重ねて確認された。

2. **SLOTS_V8 内部での完全な頑健性（E-2 修正版から。変更なし）**
   slot 定義の ±1 文字変更（V1〜V3）では Focus State（S2/S5）・方向性（B-start >> S-head）・Fisher p 値の有意性が全く変化しない。

3. **hypothesis/08 の「逆転」結果の訂正（変更なし）**
   hypothesis/08 E-2 の k=7 「逆転」は first-fit アルゴリズムバグによる人工産物であり、正しい貪欲分割では発生しない。

4. **語集合選択の独立した寄与（E-3 で新たに確認）**
   複合語集合は単独語プールからの任意選択と比べて HMM 的に異質であることが 3/4 モデルで確認された。特に trigram k=8 では単独語ランダムサンプルとの差が 778× と極大であり、S5 状態の B-start 特化が複合語集合の本質的な性質であることが示された。Bigram k=7 は E-1・E-3 ともに棄却失敗となり、このモデルのシグナルは HMM 状態空間構造のみに由来することが確定した。

5. **E-3 は既存結論を否定しない。精緻化・補強する。**
   E-3 は E-1・E-2 で得られた「境界シグナルの実在性と頑健性」を否定しない。「境界位置の寄与（E-1）」と「語集合の寄与（E-3）」は独立した効果として共存する。両方の寄与が確認されたモデル（trigram）では、境界シグナルの解釈が「境界位置の正確さ」と「複合語語クラスの特殊性」の両面から支持される。
