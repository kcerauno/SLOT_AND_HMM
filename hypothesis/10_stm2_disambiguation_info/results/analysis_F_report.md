## 情報量の解釈

- **H(label | s_t=Focus)**: Focus State 出現時点での境界ラベルの不確実性（ビット）
  - 1.0 bit = B-start/B-end が 50/50 で完全に不確定
  - 0.0 bit = 一方のみ（完全に確定済み）
- **MI = I(label; s_{t-2} | s_t=Focus)**: s_{t-2} が解消できる不確実性の量
  - 値が大きいほど s_{t-2} による曖昧性解消が有効
- **acc_with - acc_without**: s_{t-2} による予測精度の向上幅

## 解釈・考察

### 1. Trigram の方が B-start/B-end の均衡が高い

Trigram の H(label) ≈ 0.96–0.98 bits（ほぼ 1.0 bit = 50/50）に対し、
Bigram の H(label) ≈ 0.80–0.84 bits（B-start に偏り）。
これは Trigram の Focus State が B-start と B-end の両方に均等に出現する汎用境界マーカーとして機能しており、
Bigram の Focus State は B-start 寄りのシグナルであることを示す。

### 2. s_{t-2} による予測精度の向上

| モデル | k | acc 向上 |
|--------|---|---------|
| Bigram  | 7 | +0.2%  |
| Bigram  | 8 | +0.8%  |
| Trigram | 7 | **+3.2%** |
| Trigram | 8 | **+4.2%** |

Trigram で向上幅が大きいのは、Trigram が s_{t-2} を遷移確率に組み込んでいるため、
Viterbi パスの s_{t-2} にラベル情報が集積されているためと解釈できる。

### 3. B-end に予測を逆転させる s_{t-2} 状態の存在

- **Trigram k=7**: s_{t-2} = S0 のとき P(B-end)=0.544 > P(B-start)=0.456 → B-end に予測
- **Trigram k=8**: s_{t-2} = S2 のとき P(B-end)=0.578 > P(B-start)=0.422 → B-end に予測
- Bigram では全 s_{t-2} で常に B-start が多数 → s_{t-2} による逆転なし

これは Trigram が「2ステップ前に特定の状態を経由した場合は B-end になりやすい」という
文脈的な境界方向性を学習していることを示す。

### 4. MI の大きさと H(label) の関係

MI は Bigram k=7 が最大（0.0897 bits）だが、
これは s_{t-2} の情報量ではなく、Bigram パスが B-start 偏重で H(label) が低い
（= s_{t-2} で解消できる残差が相対的に大きく見える）ためである。
実際の acc_with - acc_without の向上幅（+0.2%）は最小で、実用的な曖昧性解消効果は最低。

**実効的な曖昧性解消という意味では Trigram k=8 が最大**（acc +4.2%、MI 0.0514 bits with H≈1 bit）。
