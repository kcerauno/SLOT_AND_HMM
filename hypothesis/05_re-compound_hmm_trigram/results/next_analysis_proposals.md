# 次フェーズ分析提案

作成日時: 2026-03-07
根拠: hypothesis/04, 05 の修正版再検証結果

---

## 背景と現状の限界

hypothesis/04（Bigram）および hypothesis/05（Trigram）の修正版再検証により、以下が確立した：

- **Focus State（B-start 特化状態）は存在する**：k=7 の S2、k=8 の S5（Bigram）/ S2, S5（Trigram）
- **B-start >> S-head**：交絡仮説は否定され、複合語境界の独立効果が確認された
- **Trigram の s_{t-2} は部分的に B-end/B-start を分離する**：k=7 で 4/7 状態が有意、k=8 で 2/8 状態が有意
- **Phantom State（縮退状態）は真に縮退**：Viterbi 占有率 0%

**現在の限界**：
1. HMM の状態が「何を表現しているか」の文字レベルの実体が不明
2. B-end 特化状態が同定されていない（B-start のみ注目）
3. 境界前後の状態遷移パターン（局所的シーケンス）が未分析
4. 基内の位置による勾配（語頭から語末に向けた状態変化）が未検証
5. V8 文法の境界定義への感度（どれだけ結果が文法依存か）が不明
6. s_{t-2} の曖昧性解消の情報量が定量化されていない

## 提案 A：放射確率行列の文字レベル分析【最優先】

### 目的

Focus State（S2, S5）および境界集中状態が「どの文字を好んで放射するか」を明らかにし、
HMM 状態の実体（音節・字種・位置的役割）を文字分布から解釈する。

### 方法

1. Bigram/Trigram モデルの放射確率行列 `emiss` (k × 32) を既存 npz から読み込む
2. 各状態について上位5文字と確率を列挙
3. Voynich 文字種（gallows, EVA の母音的/子音的文字、bench など）との対応を確認
4. V8 スロット定義（SLOTS_V8 の slot 番号と許容文字）と照合
5. Focus State の放射分布が V8 スロットの先頭スロットに集中しているかを検定

### 期待される成果

- 「S2 は gallows 系文字を放射しやすい → V8 スロット1相当」など、状態の語彙的実体が判明
- 複合語境界で B-start 特化状態が現れる理由が文字レベルで説明できるようになる

### 規模感

- 新規計算なし（既存 npz から行列読み出しのみ）
- 可視化スクリプト 1 本（100–200 行程度）

### 参照データ

- `hypothesis/03_trigram/results/hmm_model_cache/trigram_k{7,8}.npz`（emiss キー）
- `hypothesis/01_bigram/results/hmm_model_cache/bigram_k{7,8}.npz`
- `hypothesis/00_slot_model/source/slot_grammar.py`（V8 スロット定義）

## 提案 B：B-end 特化状態の同定【優先度：高】

### 目的

B-start に対応する B-end 特化状態を体系的に同定し、
複合語境界に非対称なシグナルが存在するかを確認する。

### 方法

1. hypothesis/05 のデコード済み Viterbi パスを再利用（または再デコード）
2. B-end 出現率が最高の状態を Focus State 同定と同じ手順で特定
3. B-end 特化状態の放射分布・s_{t-2} 文脈分布を B-start 特化状態と比較
4. 非対称性の統計検定（対応 Fisher 検定または chi-square）

### 期待される成果

- 境界前後で異なる状態が特化しているなら「語境界の方向性」がモデルに符号化されていると解釈できる
- B-end/B-start が同じ状態であれば「境界検出器」型の解釈が成立する

### 規模感

- hypothesis/05 のスクリプトを小幅拡張（50–100 行追加）
- 追加デコード不要（既存パスを保存して再利用すれば理想的）

## 提案 C：境界通過時の状態遷移パターン分析【優先度：高】

### 目的

複合語境界の直前・直後（e.g., B-end-1, B-end, B-start, B-start+1）での状態シーケンスに
規則的なパターンがあるかを調べる。

### 方法

1. 複合語の各境界位置を中心に ±2 位置の Viterbi 状態列を収集
2. 境界前後の (s_{-2}, s_{-1}, s_0, s_{+1}, s_{+2}) 5-gram 頻度を集計
3. 上位パターンと単独語同位置（語中央 ±2）のパターンを比較
4. 特異的な境界通過パターンをカイ二乗検定で検出

### 期待される成果

- 「境界前は状態 A → B、境界後は C → D」のような定型遷移が発見できれば、
  HMM が複合語構造を内部的に表現していることの強い証拠となる
- Trigram と Bigram で遷移パターンの鮮明さを比較できる

### 規模感

- 新規スクリプト（150–200 行程度）
- 既存のデコード結果（各語の Viterbi パス）を保存したファイルがあれば計算不要

## 提案 D：基内位置勾配分析【優先度：中】

### 目的

複合語の各基（base）内部で、語頭から語末に向けて状態分布がどう変化するかを分析し、
HMM 状態が「基内の位置的役割」を符号化しているかを検証する。

### 方法

1. 各基について「語頭からの相対位置」と「語末からの相対位置」を計算（正規化）
2. 相対位置ごとの状態分布を集計
3. 相対位置 0（先頭）から 1（末尾）にかけての各状態出現率の変化を可視化
4. 線形回帰（位置 → 状態出現率）の傾きで勾配強度を定量化

### 期待される成果

- 「語頭では状態 X が多く、語末では状態 Y が多い」という勾配が確認されれば、
  V8 スロット構造の反映としての解釈が強化される
- 勾配の有無が複合語と単独語で異なるかを比較検証できる

### 規模感

- 新規スクリプト（150–200 行程度）
- 短い基（2文字）が多いため、正規化の定義を慎重に設計する必要あり

## 提案 E：V8 文法感度分析・置換検定【優先度：中、妥当性確認として重要】

### 目的

現在の「複合語」定義は V8 文法（SLOTS_V8 の正規表現マッチ）に依存している。
分析結果がこの文法定義にどれだけ依存しているか（頑健性）を検証する。

### 方法

#### E-1: 置換検定（境界ラベルのシャッフル）

1. compound_words.txt の境界位置をランダムにシャッフル（同じ語長の語内で）
2. シャッフル後の境界ラベルで hypothesis/05 の B-start/B-end 分析を実行
3. 1,000 回繰り返して Null 分布を構築
4. 実際の Fisher p 値が Null 分布の上位 5% に入るかを確認

#### E-2: 文法バリアント感度分析

1. SLOTS_V8 の主要スロット定義を小幅に変更（±1 文字許容など）
2. 変更後の compound_words.txt を再生成
3. hypothesis/05 と同じ分析を実行し Focus State が同定されるかを確認

### 期待される成果

- 置換検定で帰無仮説が棄却されれば、現在の境界シグナルが V8 文法定義に依存しないことが示される
- 文法バリアント分析で結果が安定していれば、解釈の頑健性が確認される

### 規模感

- E-1: 置換スクリプト 1 本（200–300 行）+ 1,000 回ループ（CPU で 10–30 分程度）
- E-2: slot_grammar.py の改変版が必要（別 hypothesis ディレクトリ推奨）

## 提案 F：s_{t-2} 曖昧性解消の情報量定量化【優先度：低】

### 目的

Trigram の s_{t-2} が B-end/B-start の役割識別にどれだけ情報をもたらすかを
情報理論的に定量化し、Bigram との差を数値で示す。

### 方法

1. Focus State が出現する位置の (s_{t-2}, label) の同時分布 P(s_{t-2}, label) を推定
2. 条件付き相互情報量 I(label; s_{t-2} | s_t = Focus) を計算
3. Bigram 相当（s_{t-2} なし）の予測精度（B-end/B-start の正解率）と Trigram の予測精度を比較
4. k=7, k=8 および Bigram k=7, k=8 の 4 条件で比較表を作成

### 期待される成果

- Trigram が Bigram に対して「境界認識」でどれだけ情報を追加しているかを定量化
- 「s_{t-2} で解消できる曖昧性は XX bits」という定量的な主張が可能になる

### 規模感

- 新規スクリプト（100–150 行程度）
- hypothesis/05 のデコード済みパスを再利用可能
