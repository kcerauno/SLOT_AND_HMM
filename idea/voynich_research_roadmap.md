## 1. 両分析の統合的位置づけ

### 方法論の相互補完

| 手法 | 種別 | アプローチ |
|---|---|---|
| スロット文法 V8 | 生成モデル | 人間が規則を設計（演繹的） |
| HMM (k=7, k=8) | 識別モデル | データから状態を自律学習（帰納的） |

**核心的意味**: 独立した2つの手法が同じ単位（e系列・i系列・ch/sh・q-o連鎖など）を収束的に発見した。これはそれらの単位がヴォイニッチ語の構造に**統計的・生成的に実在する**ことの相互検証になっている。

### 主要な対応関係

| HMM状態 | スロット文法 V8 | 対応内容 |
|---|---|---|
| S1/S6: C-onset `ch`/`sh` 語頭 | Slot 5 `ch` `sh` | 語頭子音クラスターとして独立同定 |
| S0: E-cluster `e`/`ee`/`eee` | Slot 7 `eee` `ee` `e` | e系列の塊が独立単位として一致 |
| S5/S7: I-run `i`/`ii`/`iii` | Slot 11 `iii` `ii` `i` | i連続が独立単位として完全一致 |
| S4/S0: O/Q-cluster `q`+`o` | Slot 1 `q` + Slot 2 `o` | q-o連鎖が構造的単位として一致 |
| S2: A-frame（語頭`a`、語末`y`） | Slot 10 `a`、Slot 14 `y` | フレーム構造がスロット位置分離に対応 |
| S6: H-coda（語中・語末の`h`系） | Slot 6 `cth`/`ckh`、Slot 8 `ch`/`sh` | 語中コーダとしての`h`クラスターが対応 |

## 2. 未解決の重要問題

### 問題A: Full HMM > L-to-R HMM の差の正体

k=7での差: Full −126,899 vs L-to-R −134,963（**差 = 8,064 nats**）

この大きな差は何を意味するのか。候補は2つ：

1. **繰り返し構造説**: `ee`、`okeoke` などの双方向繰り返しが原因
2. **RTL転記説**: 右から左の原言語がLTRに転記されたことで生じた構造破壊の痕跡

→ これを判別することが次のステップの中心課題

### 問題B: Phantom State（縮退状態）の原因

k=7のS3、k=8のS4で全単語の占有率が0%。原因候補：

- BOS/EOS位置専用の遷移ハブとして機能している
- モデルがk=7〜8に対して若干過剰パラメータである
- スロット文法の「ベース連結境界」と関連している可能性

### 問題C: RTL方向性の問題

Parisel (2025) の研究では、ヴォイニッチのnグラム予測精度がRTL方向で73.9%（4グラム）に達し、ヘブライ語(61.0%)・アラビア語(64.8%)を上回ることが示されている。これはHMMの結果とどう整合するか？

## 3. 研究アイデア一覧

### 🔴 最優先：両手法の「接合点」を攻める

#### アイデア1: スロット境界 × HMM状態遷移の照合

**目的**: Phantom StateがV8の「ベース連結境界」のハブであるという仮説を直接検証  
**手法**:
- V8で複数ベースに分割される単語（例: `tazain` = `ta` + `zain`）を抽出
- その分割点でViterbiパスがどの状態遷移を示しているかを集計
- 状態切り替えタイミングとスロット境界の位置一致率を計算

**期待される知見**:
- Phantom StateがベースとベースのつなぎめにのみHighな遷移確率を持つなら → Phantom State = 境界マーカー説が証明される
- 一致率が高ければ → HMMがスロット境界を「発見」していたことになる

---

#### アイデア2: スロット番号を観測シンボルとするHMM再学習

**目的**: 「スロット間の遷移確率」を得て文法規則を確率化する  
**手法**:
- 全単語をV8でパースし、スロット列（例: `s1→s2→s7→s12→s14`）に変換
- このスロット番号列を新しい観測アルファベットとしてHMMを再学習
- 状態遷移行列からスロット間の統計的依存関係を抽出

**期待される知見**:
- どのスロットがどのスロットに続きやすいかの定量化
- 語頭・語末スロットに統計的バイアスがあるかどうかの確認
- V8が「生成できるか否か」のブール文法から「確率的な文法」への拡張

---

### 🟠 RTL仮説の直接検証

#### アイデア3: RTL HMM vs LTR HMM の対数尤度比較

**目的**: Full HMMの優位性の原因（繰り返し構造 vs RTL転記）を判別する  
**手法**:
- 全単語を逆順にしてHMMを学習（R-to-L HMM）
- LTR / RTL / Full の3モデルの対数尤度を比較

```
期待される結果パターン:

RTL ≈ LTR << Full  → 双方向性は繰り返し構造が原因
RTL > LTR << Full  → RTL構造が優位（Pariselと一致）
RTL ≈ Full > LTR   → 元の言語はRTLだった可能性が高い
```

**得られる新知見**:
- `RTL > LTR` の場合、HMMがParisel (2025) のnグラム分析を独立して検証したことになる
- Full HMMのループ遷移がRTL→LTR転記の「痕跡」であるという仮説の支持/棄却

---

#### アイデア4: Full HMM 遷移行列の方向分析

**目的**: Full HMMが学習したループ遷移の方向的偏りを定量化  
**手法**:
- 学習済み遷移確率行列 A から以下を計算：
  - 「前進遷移」(状態i → j, j > i) の確率総和
  - 「後退遷移」(状態i → j, j < i) の確率総和
- 後退遷移の偏りを統計的に検定

**期待される知見**:
- 後退遷移が有意に多い → RTL→LTR転記によるループの可能性
- ほぼ等しい → 繰り返し構造（`ee`など）が主因

---

#### アイデア5: 逆順単語をスロット文法に通す

**目的**: V8スロット文法がLTR前提の設計かどうかを検証  
**手法**:
- 全8078語を逆順にする（例: `chedy` → `ydeehc`）
- 逆順単語群に対してV8文法のカバレッジを計算
- 元の順序と逆順のカバレッジを比較

```
カバレッジが向上 → 元の言語はRTLだった可能性
カバレッジが低下 → LTR構造がスロット文法に正しく吸収されている
```

**実装コスト**: 既存のV8パーサーに逆順変換を1行追加するだけ

### 🟡 構造の縦断的検証

#### アイデア6: セクション別スロット分布の比較

**目的**: ヴォイニッチのセクション（ハーバル・星図・薬草・レシピ等）で語彙の構造が変わるかを検証  
**手法**:
- セクションごとにV8スロットの使用頻度分布を計算
- HMM状態の初期確率もセクション別に推定し直す
- セクション間でスロット使用率に有意差があるかを検定

**期待される知見**:
- 特定スロットの使用率がセクション間で有意に違えば → 語彙の異なる使い分けの証拠
- 差がなければ → 「同じ文法で異なる内容」という構造
- 「同じ文法レベルから違う」のか「語彙だけ違う」のかを判別できる

---

#### アイデア7: スロット位置のエントロピー分析

**目的**: 各スロットの「機能的強制力」を定量化する  
**手法**:
- 各スロット位置で使用された候補の頻度から Shannon エントロピーを計算
- 低エントロピー = 文法的制約が強い（機能語的）
- 高エントロピー = 自由度が高い（語根・内容語的）

**期待される知見**:
- スロット列全体のエントロピープロファイルが「左端と右端が低く、中央が高い」なら → 語頭・語末フレーム構造の定量的証明
- Slot 14（`y`のみ）のエントロピー≒0は語末接辞としての機能の証拠

### 🔵 言語仮説の検証

#### アイデア8: 既知言語とのHMM状態構造比較

**目的**: ヴォイニッチの状態構造と最も類似した言語を特定する  
**比較対象言語（優先順位順）**:

| 優先度 | 言語 | 理由 |
|---|---|---|
| 最高 | **ランダム生成文字列** | 対照群として必須。状態分化がランダム列で起きないことを証明 |
| 高 | **中世ラテン語** | 比較ベースライン。写本文化の主要言語 |
| 高 | **中世ヘブライ語** | Hauer & Kondrak (2016)、Parisel (2025) による統計的示唆 |
| 中 | **古イタリア語（トスカーナ方言）** | 写本制作地（北イタリア説）と時代的一致 |
| 中 | **アラビア語** | RTL言語として構造比較に有用 |

**比較指標**:
- Full HMM vs L-to-R HMM の対数尤度差
- 状態数k=7での状態解釈の類似性
- Phantom State（縮退状態）の出現有無

#### アイデア9: スロット文法の確率的拡張（PCFG化）

**目的**: V8を「生成できるか否か」のブール文法から確率文法へ拡張  
**手法**:
- 各スロットで各候補が選択される確率を実データから推定
- 出現頻度上位語と下位語でどのスロット選択確率が異なるかを特定

**期待される知見**:
- 低確率でしか生成されない語 → **固有名詞・専門用語候補**として浮上
- 確率分布が均一なスロット → 語彙的多様性が高い（内容語）
- 確率が極端に偏るスロット → 文法的制約が強い（機能語・接辞）

## 5. RTL仮説が正しかった場合の統合的解釈

```
RTL_HMM ≈ LTR_HMM << Full_HMM
かつ
RTL方向のnグラム予測精度 > LTR方向  [Parisel 2025]
```

この結果が得られた場合、以下のことが同時に示される：

1. **表面構造（単語内の文字配置）はLTR的**
   → スクライブはLTRに書いた（転記済み）

2. **長距離構造（文・段落レベル）はRTL的**
   → 原言語の語順・依存関係はRTLに残存

3. **Full HMMのループはその「接合面」を捉えている**
   → 転記によって生じた局所的な双方向性の痕跡

**重要性**: これはヴォイニッチ研究において「単語レベルの分析」と「文レベルの分析」がなぜ長年矛盾した結論を出し続けてきたかの**統一的な説明**になりえる。
